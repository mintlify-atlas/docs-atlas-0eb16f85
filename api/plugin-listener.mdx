title: plugin-listener
description: Subscribe to editor lifecycle events and content changes in Milkdown using the listener plugin.
---

The `@milkdown/plugin-listener` package exposes a `ListenerManager` that lets you subscribe to editor lifecycle events — mount, update, focus, blur, selection change, and destroy — as well as reactive markdown output.

## Install

<CodeGroup>

```bash npm
npm install @milkdown/plugin-listener
```

```bash yarn
yarn add @milkdown/plugin-listener
```

```bash pnpm
pnpm add @milkdown/plugin-listener
```

</CodeGroup>

## Usage

```typescript
import { Editor } from '@milkdown/kit/core'
import { commonmark } from '@milkdown/kit/preset/commonmark'
import { listener, listenerCtx } from '@milkdown/kit/plugin/listener'

const editor = await Editor.make()
  .config((ctx) => {
    ctx.get(listenerCtx).markdownUpdated((ctx, markdown, prevMarkdown) => {
      if (markdown !== prevMarkdown) {
        console.log('content changed:', markdown)
      }
    })
  })
  .use(commonmark)
  .use(listener)
  .create()
```

---

## `listener`

The Milkdown plugin. Register it with `.use(listener)` to activate event tracking.

```typescript
import type { MilkdownPlugin } from '@milkdown/ctx'

export const listener: MilkdownPlugin
```

---

## `listenerCtx`

A slice (`$ctx`) that holds the active `ListenerManager` instance. Retrieve it with `ctx.get(listenerCtx)` inside `.config()` or `editor.action()`.

```typescript
import { createSlice } from '@milkdown/ctx'

export const listenerCtx: SliceType<ListenerManager, 'listener'>
```

---

## `ListenerManager`

The class that manages all event subscriptions. Each method registers a callback and returns `this` so calls can be chained.

```typescript
export class ListenerManager {
  get listeners(): Subscribers

  beforeMount(fn: (ctx: Ctx) => void): this
  mounted(fn: (ctx: Ctx) => void): this
  updated(fn: (ctx: Ctx, doc: ProseNode, prevDoc: ProseNode | null) => void): this
  markdownUpdated(fn: (ctx: Ctx, markdown: string, prevMarkdown: string) => void): this
  blur(fn: (ctx: Ctx) => void): this
  focus(fn: (ctx: Ctx) => void): this
  destroy(fn: (ctx: Ctx) => void): this
  selectionUpdated(fn: (ctx: Ctx, selection: Selection, prevSelection: Selection | null) => void): this
}
```

### Methods

<AccordionGroup>
  <Accordion title="beforeMount(fn)">
    Called before the editor DOM is mounted. Useful for injecting context values that must be ready before the editor renders.

    ```typescript
    ctx.get(listenerCtx).beforeMount((ctx) => {
      console.log('editor about to mount')
    })
    ```
  </Accordion>

  <Accordion title="mounted(fn)">
    Called after the editor view is fully initialized and the DOM is ready.

    ```typescript
    ctx.get(listenerCtx).mounted((ctx) => {
      console.log('editor mounted')
    })
    ```
  </Accordion>

  <Accordion title="updated(fn)">
    Called when the ProseMirror document changes. The callback receives the current `doc` node and the previous `doc` node.

    Calls are debounced by 200 ms.

    ```typescript
    ctx.get(listenerCtx).updated((ctx, doc, prevDoc) => {
      console.log('doc changed', doc.nodeSize)
    })
    ```
  </Accordion>

  <Accordion title="markdownUpdated(fn)">
    Called when the document changes, with the serialized markdown string. Like `updated`, it is debounced by 200 ms.

    ```typescript
    ctx.get(listenerCtx).markdownUpdated((ctx, markdown, prevMarkdown) => {
      if (markdown !== prevMarkdown) {
        saveToServer(markdown)
      }
    })
    ```
  </Accordion>

  <Accordion title="blur(fn)">
    Called when the editor loses focus.

    ```typescript
    ctx.get(listenerCtx).blur((ctx) => {
      console.log('editor blurred')
    })
    ```
  </Accordion>

  <Accordion title="focus(fn)">
    Called when the editor gains focus.

    ```typescript
    ctx.get(listenerCtx).focus((ctx) => {
      console.log('editor focused')
    })
    ```
  </Accordion>

  <Accordion title="destroy(fn)">
    Called before the editor is torn down. Use this to clean up side effects.

    ```typescript
    ctx.get(listenerCtx).destroy((ctx) => {
      console.log('editor destroyed')
    })
    ```
  </Accordion>

  <Accordion title="selectionUpdated(fn)">
    Called whenever the ProseMirror selection changes. The callback receives the current `Selection` and the previous `Selection` (or `null` on first call).

    ```typescript
    ctx.get(listenerCtx).selectionUpdated((ctx, selection, prevSelection) => {
      console.log('selection from:', selection.from, 'to:', selection.to)
    })
    ```
  </Accordion>
</AccordionGroup>

## `Subscribers`

The shape of all registered listener arrays, accessible via `listenerManager.listeners`.

```typescript
export interface Subscribers {
  beforeMount: ((ctx: Ctx) => void)[]
  mounted: ((ctx: Ctx) => void)[]
  updated: ((ctx: Ctx, doc: ProseNode, prevDoc: ProseNode) => void)[]
  markdownUpdated: ((ctx: Ctx, markdown: string, prevMarkdown: string) => void)[]
  blur: ((ctx: Ctx) => void)[]
  focus: ((ctx: Ctx) => void)[]
  destroy: ((ctx: Ctx) => void)[]
  selectionUpdated: ((ctx: Ctx, selection: Selection, prevSelection: Selection | null) => void)[]
}
```

## `key`

The ProseMirror `PluginKey` for the internal listener state plugin, exported as `key` with the identifier `'MILKDOWN_LISTENER'`.

---

## Chaining listeners

All `ListenerManager` methods return `this`, so you can chain them:

```typescript
ctx
  .get(listenerCtx)
  .mounted((ctx) => console.log('mounted'))
  .markdownUpdated((ctx, md) => console.log(md))
  .blur((ctx) => console.log('blur'))
```
