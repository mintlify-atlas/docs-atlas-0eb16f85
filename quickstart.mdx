This guide walks you through creating a working Milkdown editor using `@milkdown/kit`. You'll install the package, mount the editor in a DOM element, load the CommonMark preset, and subscribe to content changes.

<Note>
  If you want a fully-featured editor with zero configuration, see [Using Crepe](/using-crepe) instead.
</Note>

## Install

<CodeGroup>

```bash npm
npm install @milkdown/kit
```

```bash yarn
yarn add @milkdown/kit
```

```bash pnpm
pnpm add @milkdown/kit
```

</CodeGroup>

`@milkdown/kit` re-exports everything from the core, presets, plugins, and prose packages. You can also install individual packages if you prefer a smaller bundle — the core packages are `@milkdown/core`, `@milkdown/preset-commonmark`, and `@milkdown/plugin-listener`.

## Create an editor

<Steps>
  <Step title="Add a root element">
    Milkdown needs a DOM element to mount into. Add one to your HTML:

    ```html
    <div id="editor"></div>
    ```
  </Step>

  <Step title="Import core utilities">
    Import the `Editor` class, the context keys for the root element and default value, and the CommonMark preset:

    ```typescript
    import { Editor } from '@milkdown/kit/core'
    import { rootCtx, defaultValueCtx } from '@milkdown/kit/core'
    import { commonmark } from '@milkdown/kit/preset/commonmark'
    ```
  </Step>

  <Step title="Configure and create the editor">
    Call `Editor.make()`, configure the root element and an optional default value, attach the `commonmark` preset, and call `.create()`:

    ```typescript
    const editor = await Editor.make()
      .config((ctx) => {
        ctx.set(rootCtx, document.getElementById('editor'))
        ctx.set(defaultValueCtx, '# Hello Milkdown\n\nStart editing here.')
      })
      .use(commonmark)
      .create()
    ```

    `.create()` returns a promise that resolves when the editor is fully initialized.
  </Step>
</Steps>

## Listen to content changes

Add the `listener` plugin to subscribe to markdown updates. Install it alongside the other imports:

```typescript
import { Editor } from '@milkdown/kit/core'
import { rootCtx, defaultValueCtx } from '@milkdown/kit/core'
import { commonmark } from '@milkdown/kit/preset/commonmark'
import { listener, listenerCtx } from '@milkdown/kit/plugin/listener'

const editor = await Editor.make()
  .config((ctx) => {
    ctx.set(rootCtx, document.getElementById('editor'))
    ctx.set(defaultValueCtx, '# Hello Milkdown')

    // Subscribe to markdown changes inside the config callback
    const listenerManager = ctx.get(listenerCtx)
    listenerManager.markdownUpdated((ctx, markdown, prevMarkdown) => {
      console.log('Markdown updated:', markdown)
    })
  })
  .use(commonmark)
  .use(listener)
  .create()
```

The `markdownUpdated` callback fires whenever the document changes, debounced to avoid excessive updates. The second argument is the current markdown string; the third is the previous value.

## Read markdown imperatively

You can also read the current markdown at any time using `editor.action`:

```typescript
import { getMarkdown } from '@milkdown/kit/utils'

const markdown = editor.action(getMarkdown())
```

## Complete example

```typescript
import { Editor } from '@milkdown/kit/core'
import { rootCtx, defaultValueCtx } from '@milkdown/kit/core'
import { commonmark } from '@milkdown/kit/preset/commonmark'
import { listener, listenerCtx } from '@milkdown/kit/plugin/listener'
import { getMarkdown } from '@milkdown/kit/utils'

const editor = await Editor.make()
  .config((ctx) => {
    ctx.set(rootCtx, document.getElementById('editor'))
    ctx.set(defaultValueCtx, '# Hello Milkdown\n\nStart editing.')

    ctx.get(listenerCtx).markdownUpdated((_ctx, markdown) => {
      console.log(markdown)
    })
  })
  .use(commonmark)
  .use(listener)
  .create()

// Read content at any time
document.getElementById('get-content')?.addEventListener('click', () => {
  console.log(editor.action(getMarkdown()))
})
```

## Destroy the editor

To unmount and clean up the editor, call `.destroy()`. Pass `true` to also clear all registered plugins:

```typescript
await editor.destroy()
```
