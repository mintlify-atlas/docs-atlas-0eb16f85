---
title: 'Presets'
description: 'Use CommonMark and GFM presets to add a full suite of markdown nodes, marks, commands, and keymaps to your Milkdown editor in one call.'
---

A **preset** is a pre-assembled array of `MilkdownPlugin` values that collectively implement a markdown dialect. Instead of registering individual schema nodes, input rules, keymaps, and commands one by one, you call `editor.use(commonmark)` (or `editor.use(gfm)`) and get a complete, working set of features.

Milkdown ships two official presets — both available as part of `@milkdown/kit` and as standalone packages.

## CommonMark preset

The `commonmark` preset implements the [CommonMark specification](https://spec.commonmark.org/). It is the baseline preset; most editors start here.

**Package:** `@milkdown/kit/preset/commonmark` (or `@milkdown/preset-commonmark`)

### Nodes

| Node | Description |
|------|-------------|
| `doc` | The root document node |
| `paragraph` | Standard paragraph |
| `heading` | Headings (`h1`–`h6`) via `#` |
| `blockquote` | Block quotes via `>` |
| `bullet_list` | Unordered list (`-` / `*`) |
| `ordered_list` | Ordered list (`1.`) |
| `list_item` | Individual list item |
| `code_block` | Fenced code block (` ``` `) |
| `image` | Inline images `![alt](url)` |
| `hardbreak` | Hard line break |
| `hr` | Horizontal rule (`---`) |
| `html` | Inline / block HTML pass-through |
| `text` | Raw text node |

### Marks

| Mark | Description |
|------|-------------|
| `strong` | Bold (`**text**`) |
| `emphasis` | Italic (`*text*`) |
| `inline_code` | Inline code (`` `code` ``) |
| `link` | Hyperlink `[text](url)` |

### What the preset includes

Internally, `commonmark` is a flat array of six composable plugin groups:

```typescript
// packages/plugins/preset-commonmark/src/index.ts
export const commonmark: MilkdownPlugin[] = [
  schema,       // registers nodes and marks into the ProseMirror schema
  inputRules,   // auto-formats on typing (e.g. "## " → heading)
  markInputRules, // auto-applies marks (e.g. **text** → bold)
  commands,     // registers toggle/create commands for each node/mark
  keymap,       // binds keyboard shortcuts (Mod-b → bold, etc.)
  plugins,      // registers auxiliary ProseMirror plugins
].flat()
```

### Using the CommonMark preset

<Steps>
  <Step title="Install">
    ```bash
    npm install @milkdown/kit
    ```
  </Step>
  <Step title="Use in your editor">
    ```typescript
    import { Editor, rootCtx, defaultValueCtx } from '@milkdown/kit/core'
    import { commonmark } from '@milkdown/kit/preset/commonmark'

    const editor = await Editor.make()
      .config((ctx) => {
        ctx.set(rootCtx, document.getElementById('editor'))
        ctx.set(defaultValueCtx, '# Hello, Milkdown!')
      })
      .use(commonmark)
      .create()
    ```
  </Step>
</Steps>

<Tip>
  The `commonmark` export is a plain array of plugins. You can spread it, filter it, or append additional plugins to it before passing to `use()`.
</Tip>

## GFM preset

The `gfm` preset extends CommonMark with [GitHub Flavored Markdown](https://github.github.com/gfm/) features.

**Package:** `@milkdown/kit/preset/gfm` (or `@milkdown/preset-gfm`)

### Additional nodes

| Node | Description |
|------|-------------|
| `table` | Table with header row |
| `table_row` | Table row |
| `table_cell` | Table cell |
| `table_header` | Table header cell |
| `task_list_item` | Checklist item `- [x] done` |
| `footnote_definition` | Footnote definition `[^1]:` |
| `footnote_reference` | Footnote reference `[^1]` |

### Additional marks

| Mark | Description |
|------|-------------|
| `strike_through` | Strikethrough (`~~text~~`) |

### What the preset includes

```typescript
// packages/plugins/preset-gfm/src/index.ts
export const gfm = [
  schema,       // extends the schema with GFM-specific nodes and marks
  inputRules,   // e.g. "| " triggers table creation
  pasteRules,   // handles pasted table content
  markInputRules, // e.g. ~~text~~ → strikethrough
  keymap,       // keyboard shortcuts for GFM features
  commands,     // toggle commands for GFM nodes/marks
  plugins,      // auxiliary ProseMirror plugins
].flat()
```

### Using the GFM preset

<Note>
  The GFM preset already includes all CommonMark nodes and marks. You do **not** need to add `commonmark` when using `gfm`.
</Note>

```typescript
import { Editor, rootCtx, defaultValueCtx } from '@milkdown/kit/core'
import { gfm } from '@milkdown/kit/preset/gfm'

const editor = await Editor.make()
  .config((ctx) => {
    ctx.set(rootCtx, document.getElementById('editor'))
    ctx.set(defaultValueCtx, '| Name | Role |\n|------|------|\n| Alice | Admin |')
  })
  .use(gfm)
  .create()
```

## Combining presets with other plugins

Presets are arrays of plugins. You can chain `.use()` calls to add additional plugins alongside a preset:

```typescript
import { Editor, rootCtx } from '@milkdown/kit/core'
import { commonmark } from '@milkdown/kit/preset/commonmark'
import { history } from '@milkdown/kit/plugin/history'
import { clipboard } from '@milkdown/kit/plugin/clipboard'
import { listener, listenerCtx } from '@milkdown/kit/plugin/listener'

const editor = await Editor.make()
  .config((ctx) => {
    ctx.set(rootCtx, document.getElementById('editor'))

    ctx.get(listenerCtx).markdownUpdated((ctx, markdown) => {
      console.log('Content changed:', markdown)
    })
  })
  .use(commonmark)
  .use(history)
  .use(clipboard)
  .use(listener)
  .create()
```

<Tip>
  The order of `.use()` calls generally does not matter — Milkdown's timer system ensures plugins run after their dependencies are ready, regardless of registration order.
</Tip>

    import { commonmark } from '@milkdown/kit/preset/commonmark'
